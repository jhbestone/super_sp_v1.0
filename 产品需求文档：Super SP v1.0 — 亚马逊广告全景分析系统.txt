 产品需求文档：Super SP v1.0 — 亚马逊广告全景分析系统

    1. 产品定位

    Super SP v1.0 是一个基于 AI 智能分类的亚马逊 SP（商品推广）广告数据分析工具。输入亚马逊广告后台导出的 Excel 报告（搜索词报告、广告位报告、推广商品报告），输出三份产物：可视化大屏、Excel 报告、JSON 数据。帮助卖家快速识别哪些关键词和 ASIN 在赚钱（需加大投入）、哪些在浪费钱（需否定/暂停），并给出具体的投放策略建议。

    目标用户：亚马逊 SP 广告卖家/运营人员，有 Excel 报告导出权限，需要快速从数千条搜索词中定位高价值词和浪费词。

    要解决的问题：
    - 广告报告数据量大（3 份报告/数万行），人工分析效率极低
    - 搜索词质量参差不齐（无效词/高ACoS词/超级词混杂）
    - 缺乏系统性的"分类→策略"闭环，运营凭经验做判断
    - 需要可视化看板支持快速决策

    当前状态：项目已在 v1.0 完成全部核心架构，本次 PRD 从零开始定义 v1.0 完整规格。



    2. 现有架构 / 目标架构


    super_sp_v1.0/
    ├── generate.py              # 主入口脚本（7步流水线）
    ├── report_loader.py         # 报告加载器（读取/清洗/合并 Excel）
    ├── trend_engine.py          # 趋势引擎（30天窗口计算）
    ├── classifier.py            # 14维度分类器（THRESHOLDS 可配置）
    ├── strategy_mapper.py       # 策略映射器（分类→4维度策略）
    ├── status_auditor.py        # 状态核查器（暂停/归档检测）
    ├── writer.py                # 结果写入器（JSON + Excel + HTML）
    ├── negative_keywords.py     # 否定关键词挖掘模块（手动/自动）
    ├── template.html            # 可视化面板模板（含KPI占位符+JS逻辑）
    ├── run.bat                  # Windows 批处理启动器
    ├── README.md                # 项目文档
    ├── data/                    # 输入目录：放3份Excel报告
    ├── output/                  # 输出目录：analysis.json / report.xlsx / dashboard.html
    └── Dashboard_Screenshots/   # 截图存档


    7步流水线（generate.py）：

    1. 加载数据（load_and_merge） → 2. 计算趋势（calculate_trends）
    → 3. 状态核查（audit_status） → 4. 14维度分类（classify）
    → 5. 策略映射（map_strategy） → 6. 去重 → 7. 输出（JSON+Excel+HTML）




    3. 功能清单

    P0 — 必须有（核心分析流水线）

    P0-1：数据加载与合并
    - 现状：需要从 data/ 目录加载 1~3 份报告
    - 要求：
      - 搜索词报告（必有）、广告位报告（可选）、推广商品报告（可选）
      - 自动检测文件类型（.xlsx / .xls / .csv）
      - 列名映射（中文→英文，如"广告活动名称"→"campaign_name"）
      - 派生指标计算：ACoS = cost/sales，CTR = clicks/impressions，CVR = orders/clicks
      - 所有数值列强制转 numeric，缺失填 0.0
    - 输出：search_df / prod_df / place_df 三个 DataFrame + search_file_name（原始文件名）

    P0-2：30天趋势计算
    - 现状：无趋势分析
    - 要求：
      - 以搜索词为主表，计算 30 天窗口内数据
      - 分割窗口：近 7 天 vs 前 23 天（共 30 天）
      - 按 search_term + campaign_name + ad_group_name 分组聚合
      - 计算指标：clicks / cost / orders / sales / impressions 的 sum + ctr / conv_rate / acos 的 mean
      - 计算趋势偏差：(近7天实际值 - 近7天期望值) / 近7天期望值，期望值 = (30天总值/30)*7
      - 返回 trends DataFrame
    - 输出：trends DataFrame（含 _recent_7d 和 _all 后缀字段，以及 _trend_dev 偏差字段）

    P0-3：14维度分类
    - 现状：纯手动分类
    - 要求：
      - 基于 THRESHOLDS 字典（target_acos=0.30, super_orders_min=10 等7个阈值）
      - 对每行数据依次判断 7 种关键词类型 + 7 种 ASIN 类型
      - 判断逻辑（优先级从高到低）：
        1. c30>=10 且 o30==0 → "无效关键词"
        2. ctr<0.5% 且 imp>100 → "低效关键词"
        3. ctr<0.2% 且 imp>50 → "无竞争力关键词"
        4. c30>=20 且 acos>45% → "高竞争关键词"
        5. o30>=10 且 acos<24% 且趋势上升 → "超级关键词"
        6. o30>=5 且 24%<=acos<=30% 且趋势上升 → "高潜力关键词"
        7. o30>=1 且 acos<39% → "次级潜力关键词"
      - ASIN 类型：在以上判断后，若 targeting 字段含 "ASIN"/"PRODUCT" 则映射为对等 ASIN 分类
    - 输出：类型列（14种值之一）

    P0-4：策略映射
    - 现状：无系统策略
    - 要求：
      - 根据分类结果 + 状态标记，生成 4 维度策略建议
      - 定向策略（建议定向策略）：
        - 超级→"积极投放"，高潜力→"积极投放"，次级→"维持投放"
        - 高竞争/低效→"谨慎/优化投放"，无效→"立即停止"
      - 竞价策略（建议竞价策略）：+20-30% / +10-20% / 维持 / -20-30% / 停止
      - 投放位置（建议投放位置）：顶部加价 / 适度加价 / 标准 / 减少展示 / 停止
      - 匹配类型（建议匹配类型）：广泛+词组 / 词组 / 当前匹配 / 精准 / 否定
      - 已暂停/已归档的对象直接输出"无需操作"
    - 输出：df 新增 5 个策略列（建议策略 / 建议定向策略 / 建议竞价策略 / 建议投放位置 / 建议匹配类型）

    P0-5：去重
    - 要求：
      - 按 campaign_name + search_term 去重
      - 保留 cost_recent_7d 最高的一条
      - 输出去重前后的行数对比
    - 输出：去重后的 DataFrame

    P0-6：三格式输出
    - 要求：
      6a. JSON (analysis.json)：
      - 8 字段简化格式：类型 / 广告组合 / 广告活动名称 / 广告组名称 / 客户搜索词/ASIN / 建议定向策略 / 匹配类型 / 建议策略洞察
      - 建议策略洞察：从建议策略字段简化为短文本（如"增加预算和溢价，优先推广"、"降低竞价测试潜力"等）

      6b. Excel (report.xlsx)：
      - Sheet1: 全量数据（含所有列，中文列名）
      - Sheet2: 汇总统计（记录数、花费、订单、ACoS 等）
      - Sheet3: 进攻型策略（超级/高潜力）
      - Sheet4: 防守型策略（无效/低效/无竞争力/高竞争）

      6c. HTML 可视化大屏 (dashboard.html)：
      - 将 JSON 数据注入 template.html 的 mockData 数组（正则替换）
      - 替换所有 {{PLACEHOLDER}} 为实际 KPI 值
      - KPI：TOTAL_COST/TOTAL_SALES/TOTAL_ORDERS/ACOS/AVG_CPC
      - 花费结构：ASIN_NO_ORDER/ASIN_HAS_ORDER/ASIN_TOTAL/KEYWORD_NO_ORDER/KEYWORD_HAS_ORDER/KEYWORD_TOTAL/KEYWORD_NO_ORDER_PCT/HIGH_CLICK_PCT/LOW_CLICK_PCT
      - 替换数据来源和生成时间

    P1 — 应该有

    P1-1：否定关键词挖掘模块（negative_keywords.py）
    - 要求：
      - 手动模式：从广告数据中提取已有的「词组否定」和「精准否定」关键词
      - 自动模式：多维度计算，自动筛选需要否定的关键词
        - 高点击无转化（clicks≥5, orders=0）
        - 低 CTR（CTR<0.5%, clicks≥10, impressions>100）
        - 高 ACoS（ACoS>100%, cost>0, sales>0）
      - 多维度筛选：
        - 出单标识：ASIN-不出单/出单，关键词-不出单/出单
        - 指标范围：花费/点击/订单/CTR/ACoS 上下限
        - 关键词搜索
      - 自动去重（按 keyword 去重，保留 cost 最高的一条）
      - 返回候选列表和汇总统计

    P1-2：可视化大屏 — KPI 实时更新
    - 要求：
      - 每次筛选后自动更新顶部 KPI 卡片
      - 5 个核心指标：总花费 / 总销售额 / 总订单数 / 整体 ACoS / 平均 CPC
      - 指标值从当前过滤后的数据实时计算
      - KPI 由 Python writer.py 的 calculate_kpis() 计算，通过 {{PLACEHOLDER}} 注入
      - 同时前端 JS 在 applyFilters() 时也重新计算并更新 DOM

    P1-3：可视化大屏 — 广告组合（Portfolio）筛选联动
    - 要求：
      - 选择广告组合时，广告活动下拉列表自动同步（只显示该组合下的活动）
      - "-"（空组合）显示为 "No Portfolio"
      - 前端维护 portfolioCampaignMap：{portfolio_name: [campaign_names]}

    P1-4：可视化大屏 — 选定广告活动指标板块
    - 要求：
      - 展示：已选广告活动数量 / 30天花费 / 30天点击次数 / 30天订单 / 30天广告销售额 / 30天转化率 / 30天ACoS
      - 数据随筛选变化实时更新
      - 转化率 = 总订单数 / 总点击量 * 100%
      - ACoS = 广告花费 / 广告销售额 * 100%

    P1-5：可视化大屏 — 花费结构分析
    - 要求：
      - 分 ASIN ~ 不出单/出单 总花费（6项）
      - 分关键词~不出单/出单 总花费
      - 分 ASIN总花费 / 关键词总花费
      - 占比数据：关键词不出单花费占比 / 高点击不出单占比（>17次点击无订单）/ 低点击不出单占比（<5次点击无订单）
      - 数据随筛选实时更新

    P2 — 锦上添花

    P2-1：可视化大屏 — 表头排序
    - 现状：无排序
    - 要求：所有 13 列均支持点击排序（asc/desc/null 三态切换），表头显示 ▲/▼ 图标

    P2-2：可视化大屏 — 分页
    - 现状：无分页
    - 要求：50 条/页，首页/上一页/页码/下一页/末页，跳转输入框，显示"共 N 条 第 X/Y 页"

    P2-3：可视化大屏 — 导出 CSV
    - 现状：无导出
    - 要求：导出当前过滤结果为 CSV，BOM 头支持 Excel 中文，字段中文列名

    P2-4：可视化大屏 — 侧栏分类统计数
    - 现状：无统计
    - 要求：14种分类 + 全部，每类显示当前过滤后的记录数 badge

    P2-5：run.bat — Windows 一键运行
    - 要求：
      - 检查 data/ 目录是否存在
      - 检查是否有 .xlsx 文件
      - 使用 Hermes Python 环境运行 generate.py
      - 运行完成后自动打开 dashboard.html



    4. 非功能要求

    - 兼容性：Python 3.10+，pandas / openpyxl 依赖
    - 性能：单次运行 < 5 分钟（万级数据量）
    - 输入：只读 data/ 目录下的 .xlsx/.xls/.csv 文件，不修改源文件
    - 输出编码：JSON 和 HTML 均为 UTF-8 编码，CSV 含 BOM
    - 调用方式：python generate.py [data_dir] [output_dir] 或 run.bat 双击
    - 前端：纯静态 HTML，无构建工具，ECharts 通过 CDN 加载（联网需求）



    5. 不做什么（Explicit Exclusions）

    - ❌ 不接入亚马逊 API（只处理本地 Excel 文件）
    - ❌ 不修改 data/ 目录下的原始报告文件
    - ❌ 不做用户登录/权限系统
    - ❌ 不做服务器端部署，纯本地运行
    - ❌ 不处理 Google Ads 或其他广告平台数据
    - ❌ 不做多语言支持（仅中文）
    - ❌ 不依赖数据回传/增量更新（每次全量重跑）
    - ❌ 不做单元测试/CI（后续版本考虑）
    - ❌ 不连接数据库，所有数据在内存中处理
